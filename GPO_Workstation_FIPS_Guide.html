🔍 Script Gap Analysis & Recommendations

Your script provides a solid baseline but is missing several items that are required to fully compensate for FIPS being disabled. The table below summarizes what you have, what is missing, and the priority of each gap.

🚨
Critical: Add AES ciphers before finishing The absence of explicit AES enablement in your script is the most significant gap. When FIPS mode is off and you disable all legacy ciphers, Windows should fall back to AES — but explicitly enabling AES 128 and AES 256 in SCHANNEL is required to guarantee this and to pass most security audits. The remediation script in Step 4 includes this fix.

Step 1 — Create a New OU for the Workstation

🖥 GUI (ADUC)
💻 PowerShell / CLI
1
Open Active Directory Users and Computers
Press Win + R, type dsa.msc, press Enter. Alternatively, open Server Manager → Tools → Active Directory Users and Computers.
2
Navigate to the desired parent container
In the left tree, expand your domain (e.g., corp.example.com). Right-click the parent OU where you want the new OU to live (e.g., the root of the domain, or an existing Workstations OU).
3
Create the new OU
Right-click → New → Organizational Unit. Name it something descriptive, e.g., FIPS-Exempt-Workstations. Check "Protect container from accidental deletion". Click OK.
4
Move the workstation computer object into the new OU
In ADUC, locate the workstation's computer object (it may be in Computers or another OU). Right-click it → Move… → select the new OU → OK.
# ── Prerequisites ───────────────────────────────────────────────────────────
# Run from an elevated PowerShell session on a DC or machine with RSAT.
# Replace values in the variables below before running.
# ────────────────────────────────────────────────────────────────────────────

# Variables
$ouName       = "FIPS-Exempt-Workstations"
$parentOU     = "OU=Workstations,DC=corp,DC=example,DC=com"   # adjust to your domain
$domainDN     = "DC=corp,DC=example,DC=com"
$workstation  = "DESKTOP-EXEMPT01"                              # computer name

# 1. Import the AD module
Import-Module ActiveDirectory

# 2. Create the new OU (protected from accidental deletion)
New-ADOrganizationalUnit `
    -Name $ouName `
    -Path $parentOU `
    -ProtectedFromAccidentalDeletion $true `
    -Description "Workstations exempt from FIPS with compensating TLS controls"

# 3. Verify the OU was created
Get-ADOrganizationalUnit -Filter { Name -eq $ouName } | Select-Object DistinguishedName

# 4. Move the workstation computer object into the new OU
$computer = Get-ADComputer -Identity $workstation
$targetOU = "OU=$ouName,$parentOU"

Move-ADObject -Identity $computer.DistinguishedName -TargetPath $targetOU

Write-Host "✔ OU created and workstation moved successfully." -ForegroundColor Green
⚠️
Note: Replace OU=Workstations,DC=corp,DC=example,DC=com with your actual domain structure. Run Get-ADDomain | Select-Object DistinguishedName to retrieve your domain DN.

Step 2 — Copy Domain GPO and Apply to Workstation OU

Rather than starting from scratch, copy your existing domain GPO into a new GPO that will be linked exclusively to the new OU. This preserves all existing domain settings as your baseline.
🖥 GUI (GPMC)
💻 PowerShell / CLI
1
Open Group Policy Management Console
Press Win + R, type gpmc.msc, press Enter. Or: Server Manager → Tools → Group Policy Management.
2
Locate the source domain GPO to copy
In the GPMC tree: Forest → Domains → [Your Domain] → Group Policy Objects. Identify the GPO you want to use as your baseline (e.g., the Default Domain Policy or a custom security GPO).
3
Copy the GPO
Right-click the source GPO → Copy. Then right-click Group Policy Objects → Paste. In the wizard: choose "Preserving existing permissions" if prompted. Name the new GPO something like FIPS-Exempt-Workstation-Policy.
4
Link the new GPO to the workstation OU
In the GPMC tree, right-click your new OU (FIPS-Exempt-Workstations) → Link an Existing GPO… → select FIPS-Exempt-Workstation-Policy → OK.
5
Set link order and enforce (optional)
In the OU's Linked Group Policy Objects tab, confirm the new policy is listed. If you want it to override parent policies, right-click the link → Enforced. For the workstation to pick up the GPO immediately, run gpupdate /force on it.
# Variables (adjust to match your environment)
$sourceGPOName = "Default Domain Security Policy"    # name of GPO to copy FROM
$newGPOName    = "FIPS-Exempt-Workstation-Policy"
$targetOU      = "OU=FIPS-Exempt-Workstations,OU=Workstations,DC=corp,DC=example,DC=com"
$domain        = "corp.example.com"

# 1. Copy the existing GPO to a new GPO
$sourceGPO = Get-GPO -Name $sourceGPOName
$newGPO    = Copy-GPO -SourceGuid $sourceGPO.Id -TargetName $newGPOName

Write-Host "✔ GPO copied. New GPO GUID: $($newGPO.Id)"

# 2. Link the new GPO to the workstation OU
New-GPLink `
    -Name $newGPOName `
    -Target $targetOU `
    -LinkEnabled Yes `
    -Enforced No     # set to Yes if you need to override parent GPOs

Write-Host "✔ GPO linked to OU: $targetOU"

# 3. Verify the link
Get-GPInheritance -Target $targetOU | Select-Object -ExpandProperty GpoLinks

Step 3 — Disable the FIPS Flag on the Workstation OU GPO

🖥 GUI (GPME)
💻 PowerShell / CLI
1
Open the GPO for editing
In GPMC, right-click FIPS-Exempt-Workstation-Policy → Edit…. This opens the Group Policy Management Editor (GPME).
2
Navigate to the FIPS policy setting
Computer Configuration → Windows Settings → Security Settings → Local Policies → Security Options
Scroll down to find:
"System cryptography: Use FIPS compliant algorithms for encryption, hashing, and signing"
3
Disable the FIPS setting
Double-click the policy → check "Define this policy setting" → select Disabled → click OK.
4
Also disable via Registry Preferences (belt-and-suspenders)
Navigate to:
Computer Configuration → Preferences → Windows Settings → Registry
Right-click Registry → New → Registry Item. Set:
• Action: Update
• Hive: HKEY_LOCAL_MACHINE
• Key Path: SYSTEM\CurrentControlSet\Control\Lsa\FipsAlgorithmPolicy
• Value name: Enabled
• Value type: REG_DWORD
• Value data: 0
⚠️
GPO precedence: If a higher-level GPO (e.g., the Default Domain Policy) enforces FIPS = Enabled, your OU-level GPO will be overridden unless you mark the OU link as Enforced. Check GPO inheritance with GPMC → [OU] → Group Policy Inheritance tab.
# ── Method 1: Set FIPS = Disabled via Group Policy Registry Preference ───────
# This sets the registry key directly through the GPO (recommended approach).

$gpoName = "FIPS-Exempt-Workstation-Policy"
$gpo     = Get-GPO -Name $gpoName

# Set the FIPS registry value inside the GPO using Set-GPRegistryValue
Set-GPRegistryValue `
    -Guid $gpo.Id `
    -Key  "HKLM\SYSTEM\CurrentControlSet\Control\Lsa\FipsAlgorithmPolicy" `
    -ValueName "Enabled" `
    -Type DWord `
    -Value 0

Write-Host "✔ FIPS registry value set to 0 in GPO."

# ── Method 2: Set via Security Policy INF (handles the Security Options node) ─
# Use this if Method 1 alone does not fully override the Security Options setting.

# Retrieve GPO's SYSVOL path and write a secedit INF
$gpoDomain = "corp.example.com"
$gpoId     = $gpo.Id
$sysvolPath = "\\$gpoDomain\SYSVOL\$gpoDomain\Policies\{$gpoId}\Machine\Microsoft\Windows NT\SecEdit"

# Create SecEdit directory if it doesn't exist
if (-not (Test-Path $sysvolPath)) { New-Item -Path $sysvolPath -ItemType Directory | Out-Null }

$infContent = @"
[Unicode]
Unicode=yes
[Version]
signature="`$CHICAGO`$"
Revision=1
[System Access]
[Registry Values]
MACHINE\System\CurrentControlSet\Control\Lsa\FIPSAlgorithmPolicy\Enabled=4,0
"@

Set-Content -Path "$sysvolPath\GptTmpl.inf" -Value $infContent -Encoding Unicode

# Increment GPO version so clients detect the change
$gpoObj = [ADSI]"LDAP://CN={$gpoId},CN=Policies,CN=System,DC=corp,DC=example,DC=com"
$version = [int]$gpoObj.versionNumber[0] + 1
$gpoObj.versionNumber = $version
$gpoObj.CommitChanges()

Write-Host "✔ Security Options INF written and GPO version incremented."

Complete Compensating Controls Script (Corrected + Supplemented)

⚠️
Deployment: Attach this as a GPO Computer Startup Script, or deploy via GPO Registry Preferences using Set-GPRegistryValue. A restart is required for SCHANNEL changes to take effect.
#Requires -RunAsAdministrator
###############################################################################
#  FIPS-Exempt Workstation — Compensating TLS/Cipher Controls
#  Version 2.0  |  Addresses all gaps identified in security review
#  Apply via: GPO Computer Startup Script or Set-GPRegistryValue
###############################################################################

function Set-SCHANNELKey {
    param($Path, $Name, $Value)
    New-Item -Path $Path -Force | Out-Null
    Set-ItemProperty -Path $Path -Name $Name -Value $Value -Type DWord
}

$base = "HKLM:\System\CurrentControlSet\Control\SecurityProviders\SCHANNEL"

# ── TLS/SSL Protocols ────────────────────────────────────────────────────────
foreach ($proto in @("SSL 2.0", "SSL 3.0", "TLS 1.0", "TLS 1.1")) {
    foreach ($role in @("Server", "Client")) {
        $p = "$base\Protocols\$proto\$role"
        Set-SCHANNELKey $p "Enabled"          0
        Set-SCHANNELKey $p "DisabledByDefault" 1  # NEW — belt-and-suspenders
    }
}

# Enable TLS 1.2
foreach ($role in @("Server", "Client")) {
    $p = "$base\Protocols\TLS 1.2\$role"
    Set-SCHANNELKey $p "Enabled"          1
    Set-SCHANNELKey $p "DisabledByDefault" 0
}

# NEW — Enable TLS 1.3 (Windows 10 1903+ / Server 2022)
foreach ($role in @("Server", "Client")) {
    $p = "$base\Protocols\TLS 1.3\$role"
    Set-SCHANNELKey $p "Enabled"          1
    Set-SCHANNELKey $p "DisabledByDefault" 0
}

# ── Ciphers: Disable Weak ────────────────────────────────────────────────────
$weakCiphers = @(
    "RC4 128/128", "RC4 64/128", "RC4 56/128", "RC4 40/128",
    "RC2 128/128", "RC2 56/128",  "RC2 40/128",    # NEW — RC2 family
    "DES 56/56", "Triple DES 168", "NULL"
)
foreach ($c in $weakCiphers) {
    Set-SCHANNELKey "$base\Ciphers\$c" "Enabled" 0
}

# NEW — Explicitly enable AES (critical FIPS compensating control)
foreach ($aes in @("AES 128/128", "AES 256/256")) {
    Set-SCHANNELKey "$base\Ciphers\$aes" "Enabled" 1
}

# ── Hashes ───────────────────────────────────────────────────────────────────
Set-SCHANNELKey "$base\Hashes\MD5"    "Enabled" 0
foreach ($h in @("SHA", "SHA256", "SHA384", "SHA512")) {  # SHA384 added
    Set-SCHANNELKey "$base\Hashes\$h" "Enabled" 1
}

# ── Key Exchange Algorithms ──────────────────────────────────────────────────
foreach ($kx in @("Diffie-Hellman", "ECDH", "PKCS")) {
    Set-SCHANNELKey "$base\KeyExchangeAlgorithms\$kx" "Enabled" 1
}

# NEW — Minimum DH key size: require 2048-bit to prevent weak DH (Logjam)
Set-SCHANNELKey "$base\KeyExchangeAlgorithms\Diffie-Hellman" "ServerMinKeyBitLength" 2048
Set-SCHANNELKey "$base\KeyExchangeAlgorithms\Diffie-Hellman" "ClientMinKeyBitLength" 2048

# ── .NET Framework Strong Crypto ─────────────────────────────────────────────
foreach ($v in @("v4.0.30319", "v2.0.50727")) {
    foreach ($hive in @("SOFTWARE\Microsoft\.NETFramework", "SOFTWARE\Wow6432Node\Microsoft\.NETFramework")) {
        Set-ItemProperty -Path "HKLM:\$hive\$v" -Name "SchUseStrongCrypto" -Value 1 -Type DWord
    }
}

# ── Cipher Suite Order ───────────────────────────────────────────────────────
# NEW — Enforce strong cipher suite preference order via registry
# (GPO Admin Template "SSL Cipher Suite Order" applies this same key)
$cipherSuites = @(
    "TLS_AES_256_GCM_SHA384",
    "TLS_AES_128_GCM_SHA256",
    "TLS_CHACHA20_POLY1305_SHA256",
    "TLS_ECDHE_ECDSA_WITH_AES_256_GCM_SHA384",
    "TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256",
    "TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384",
    "TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256",
    "TLS_DHE_RSA_WITH_AES_256_GCM_SHA384",
    "TLS_DHE_RSA_WITH_AES_128_GCM_SHA256"
) -join ","

New-Item -Path "HKLM:\SOFTWARE\Policies\Microsoft\Cryptography\Configuration\SSL\00010002" -Force | Out-Null
Set-ItemProperty `
    -Path  "HKLM:\SOFTWARE\Policies\Microsoft\Cryptography\Configuration\SSL\00010002" `
    -Name  "Functions" `
    -Value $cipherSuites `
    -Type  String

Write-Host "✔ All compensating TLS/cipher controls applied. A restart is required." -ForegroundColor Green
💡
To deploy via GPO Registry Preferences instead of a startup script, use Set-GPRegistryValue for each key/value pair, substituting your GPO GUID. This avoids the need for startup script execution policy configuration and applies changes as a proper GPO setting.
